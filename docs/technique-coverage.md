## FIXME 付き (荒い heuristic、約 20 件)

実装はしているが文脈不足 / 近似のため将来 tighten 対象。実戦で誤判定気づいたら個別修正。

### 連続手必要 (単発判定では `false` 固定)
- `継ぎ歩` — 直前手も同じ筋への歩打が必要
- `連打の歩` — 同様、複数手前まで遡る必要

### 「取られる」判定が近似
- `角切り` / `飛車切り` / `馬切り` / `龍切り` — 駒を取らせる手筋。現状は「移動先が相手の利きにある」で代用、自陣の駒で守られているか等は見ない
- `成捨て` / `タダ捨て` — 同様

### 配置近似
- `たたきの歩` — 前方 1 マスに相手駒があるかで判定、形勢評価は無し
- `底歩に香` — 「同筋に底歩 + その上に香」の 1 手構築のみ
- `桂頭の玉` — 玉が自桂の前へ寄った形を簡易判定
- `玉頭戦` — 玉周囲 2 マス以内への攻撃手を簡易判定
- `卓上の銀` — 中央 4-6 筋 4-6 段への銀打を簡易判定
- `玉単騎` — 玉周囲 1 マス自駒なし状態での玉移動
- `三桂懐刃` — 自分の桂 + 成桂が合計 3 枚以上
- `跳ね違いの桂` — 「右桂は左へ / 左桂は右へ」を file 位置で判定
- `継ぎ桂` — 直前手で同筋桂跳ね済前提
- `吊るし桂` — 相手玉の 2 マス前後 ± 1 筋に桂打
- `銀ばさみ` — 動かした駒の利きに相手銀 + 反対側に自駒利き
- `たすきの銀` — 斜め移動 + promotion zone を含む
- `幽霊角` — 自分が王手状態で角打
- `雪隠詰め` — 王手 + 隅マス到達の簡易判定
- `都詰め` — 棋書出典が揺れ、tentative

### 名前の解釈が複数
- `パンティを脱ぐ` — 「右金 (4九 or 5九) を 1 段上がる」と暫定解釈

## 改善の手段

将来的に拡張する場合の方針:

1. **連続手対応**: `TechniqueTemplate.matches(Record record, int ply)` に変更し棋譜履歴アクセス可能化 → `継ぎ歩` `連打の歩` `継ぎ桂` 等が実装可能
2. **「取られる」判定の厳密化**: `Position.hasPower(square, attacker)` を活用して「成捨てが本当に成立してるか」を判定 → `角切り` 系を tighten
3. **配置系を castle.dart に移動**: `双玉接近` `穴熊の姿焼き` `2段ロケット` 等は castle template として書き直し、本ファイルから削除
4. **手筋 DSL**: ASCII codegen と同様、手筋パターンを宣言的に書ける format を設計し codegen 化

これらは別 PR で対応予定。実戦使用で「これ誤検出 / 拾えてない」と気づいたら issue として記録してから着手する。
